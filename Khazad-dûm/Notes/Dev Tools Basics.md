# Dev Tools Basics

## Panel Elements

1. **Elements Panel**:
    
    - The Elements panel allows you to inspect and manipulate the HTML and CSS of a web page.
    - You can view the HTML structure of the page, including the DOM tree, and modify styles, attributes, and content in real-time.
    - It provides features like element highlighting, DOM breakpoints, and box model visualization.
      
2. **Console Panel**:
    
    - The Console panel is a JavaScript console where you can execute JavaScript code, log messages, and debug JavaScript errors and exceptions.
    - You can use it to interactively test JavaScript expressions, log debug messages, and inspect objects and variables.
    - It also displays network requests, warnings, and errors generated by the browser and JavaScript engine.
      
3. **Sources Panel**:
    
    - The Sources panel allows you to debug JavaScript code and analyze network requests.
    - You can set breakpoints, step through code execution, inspect variables, and analyze call stacks.
    - It provides features like source mapping, async stack traces, and live editing of JavaScript and CSS files.
      
4. **Network Panel**:
    
    - The Network panel shows network activity for the current page, including HTTP requests and responses, request headers, response headers, and timings.
    - You can inspect individual requests, filter requests by type or domain, and analyze network performance metrics like latency, transfer size, and resource timing.
      
5. **Performance Panel**:
    
    - The Performance panel allows you to analyze the runtime performance of a web page or application.
    - You can record performance profiles, analyze CPU usage, memory consumption, and rendering performance, and identify performance bottlenecks and optimizations.
    - It provides features like timeline recording, frame rate analysis, and memory allocation tracking.
      
6. **Application Panel**:
    
    - The Application panel provides tools for inspecting and debugging web application resources, such as local storage, session storage, cookies, and service workers.
    - You can view and modify storage data, clear caches, debug service worker scripts, and manage application manifests.
      
7. **Security Panel**:
    
    - The Security panel provides information about the security of the current page, including SSL certificate details, insecure content warnings, and security vulnerabilities.
    - You can inspect security indicators, view certificate information, and diagnose security issues affecting the page.
      
8. **Audits Panel**:
    
    - The Audits panel allows you to run automated audits on a web page to identify performance, accessibility, SEO, and best practices issues.
    - You can generate audit reports, view recommendations for improving page performance and accessibility, and track progress over time.
