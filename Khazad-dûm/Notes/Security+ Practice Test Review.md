# Security+ Practice Test Review

- **Data Processor** - a third-party organization processes personal data on behalf of a data controller, without owning or controlling the data (Ex. external payroll company).
- **Data Controller** - a data controller is an entity that determines the purposes and means of processing personal data (Ex. internal payroll company).
- **Data Owner** - A data owner is a senior-level executive or manager responsible for a specific information asset or dataset within an organization (Ex. VP of Human Resources).
- **Data Subject** - a data subject refers to an individual whose personal data is collected, stored, or processed by an organization (Ex. customer).
- **Known Environment** - a penetration tester has complete knowledge of the target system’s architecture, software, and configurations. This includes if they have full access to the environment documentation.
- **Stakeholder** - someone who will be affected by a process or change (Ex. Security testing, change management).
- **NAT** - Network address translation assigns a public address to a group of computers or devices within a private network, allowing them to access the Internet while hiding their internal private IP addresses.
- **SRTP** - Secure real-time transport protocol is a security layer that protects real-time communication and media, such as audio and video, as they travel over the internet. It achieves this by encrypting the data, making it unreadable to anyone except the intended recipient (Ex. VoIP communication).
- **SNMPv3** - Simple Network Management Protocol version 3 is a network management protocol that provides secure, reliable data about network devices. It is a significant upgrade from earlier versions of SNMP, offering enhanced security features and improved management capabilities.
- **DNSSEC** - Domain name system security extensions is a set of Internet Engineering Task Force (IETF) specifications for securing certain kinds of information provided by the Domain Name System (DNS) as used on Internet Protocol (IP) networks. It provides authentication of DNS data, authenticated denial of existence, and data integrity, but not availability or confidentiality.
- **Right to be Forgotten** - The Right to be Forgotten is a legal concept that grants individuals the right to request the removal of their personal data from publicly accessible search engines and online directories, under certain circumstances.
- **Right of Notice** - The right of notice says that data controllers must inform data subjects about data processing activities. 
- **Right of Correction** - The right of correction says that data controllers must correct any errors in PII. 
- **Right of Consent** - The right of consent says that data subjects must consent to data collection and processing. 
- **Screened Subnet** - a network security architecture that provides an additional layer of protection between internal and external networks. It involves dividing the network into three logical subnets: External Router, Perimeter Network, Internal Router. 
- **ACL** - an access control list is a list of permissions associated with a system resource (object or facility) that specifies which users or system processes are granted access to those resources, as well as what operations are allowed on given resources.
- **RSTP** - Rapid Spanning Tree Protocol is a network protocol designed to prevent network loops and ensure high availability in Ethernet networks.
- **Bluesnarfing** - a type of cyberattack that targets Bluetooth devices to gain unauthorized access and steal sensitive information. It involves hackers detecting an active Bluetooth connection and exploiting vulnerabilities to hijack the connection (Ex. stealing data on your phone).
- **Bluejacking** - an attack where an individual sends unsolicited messages to a Bluetooth-enabled device within a certain range. This type of attack does not involve device hijacking or control; instead, the attacker sends anonymous messages, such as spam or phishing attempts, to the targeted device.
- **WPS attack** - a protocol designed to simplify the process of configuring secure wireless networks, particularly for home users and small offices. However, its convenience comes at the cost of security. WPS is vulnerable to various attacks such as brute-force attacks.
- **Assessment** - An informal review, not a formal examination such as an internal audit. 
- **DLP** - Data Loss Prevention is a cybersecurity term that refers to defending organizations against both data loss and data leakage prevention.
- **HSM** - A Hardware Security Module is a dedicated cryptographic processor designed to protect highly critical and sensitive cryptographic keys and assets.
- **COPE** - an IT business strategy where an organization purchases and provides computing resources and devices to employees, allowing them to use these devices for both personal and business purposes.
- **SED** - A Self-Encrypting Drive is a type of hard drive or solid-state drive (SSD) that automatically and continuously encrypts data as it is written to the drive, without requiring user interaction.
- **PaaS** - Platform as a service is a cloud computing model where a third-party provider delivers hardware and software tools to users over the internet (Ex. AWS Lambda, Wasabi).
- **IaaS** - Infrastructure as a service are virtualized computing resources, virtual networking, virtual storage, and virtual machines accessible over the internet (Ex. AWS). 
- **Xaas** - Anything as a service is the delivery of various security-related services over the internet, often on a subscription basis. It is an encompassing term for the aas model.
- **TPM** - A Trusted Platform Module is a dedicated hardware-based security microcontroller integrated into a computer’s motherboard or processor. It employs cryptographic techniques to securely store critical information
- **FPGA** - A Field-Programmable Gate Array is a type of configurable integrated circuit that can be repeatedly programmed after manufacturing.
- **NIPS** - A Network-Based Intrusion Prevention System is a network security tool that continuously monitors network traffic for malicious activity and takes proactive measures to prevent or block detected threats. 
- **SCADA** - A (Supervisory Control and Data Acquisition) system is a control system architecture comprising computers, networked data communications, and graphical user interfaces for high-level supervision of machines and processes. It involves monitoring and controlling processes from a central location, making it susceptible to cyber threats.
- **ICS** - Industrial Control Systems refer to the integration of hardware and software with network connectivity to support production processes in manufacturing or critical infrastructure (Ex. Nuclear Power Plant Systems).
- **Zero-day exploit** - is a type of cyber attack that takes advantage of a previously unknown or unaddressed security flaw (vulnerability) in computer software, hardware, or firmware. The term “zero-day” refers to the fact that the vendor or developer has only just learned of the flaw – which means they have “zero days” to fix it.
- **Buffer overflow** - a type of cyber attack where an attacker intentionally sends more data (including a long series of unusual characters) to a buffer (a temporary storage area) than it is designed to hold. This excess data overflows into adjacent memory areas allowing the attacker to exploit the device. 
- **Continuous integration** - a software development practice that ensures frequent integration of code changes into a shared repository, followed by automated verification through builds and tests. In the context of cybersecurity, CI plays a crucial role in maintaining the integrity and security of software systems.
- **Threat Hunting** - Threat hunting is a proactive cybersecurity technique where skilled analysts, known as threat hunters, scour networks, systems, and devices for anomalies to identify and mitigate cyber threats before they cause harm. Threat hunters also act on security advisories and perform threat hunting based on its information and recommendations. 
- **Tunneling Certificate** - Certificate-based authentication is a widely used mechanism for securing VPN connections. In the context of tunneling, certificates play a crucial role in establishing and maintaining secure connections. It Assists in providing authentication, authorization, and accounting (AAA) services. 
- **Tcpdump** - a command-line packet analyzer software utility that captures and displays network traffic packets in real-time. 
- **Rogue AP** - A rogue access point is a wireless access point (AP) installed on a secure network without explicit authorization from a local network administrator.
- **Evil Twin** - An evil twin is a fraudulent Wi-Fi access point that appears to be legitimate but is set up to eavesdrop on wireless communications. It is a type of man-in-the-middle (MitM) attack, where an attacker creates a fake Wi-Fi network with a similar name (SSID) and settings to a genuine network, tricking users into connecting to the fake one instead.
- **BCP** - A Business Continuity Plan in cybersecurity refers to a predetermined set of instructions or procedures that describe how an organization’s mission/business processes will be sustained during and after a significant disruption, including cybersecurity incidents.
- **BIA** - A Business Impact Analysis in cybersecurity is a critical component of business continuity planning and risk management. It evaluates the potential risks and consequences of a cybersecurity breach or disruption to an organization’s operations, data, and assets.
- **MTBF** - Mean Time Between Failures refers to the average time elapsed between two consecutive failures of a system, component, or process. It measures the reliability and availability of a system, providing a quantitative metric for evaluating its robustness.
- **RPO** - The Recovery Point Objective in cybersecurity refers to the maximum acceptable amount of data loss that an organization can tolerate after an unexpected disaster event involving data loss.
- **MTTR** - Mean Time to Remediate measures the average time it takes for an organization to respond to and remediate a detected security threat or incident. It represents the time elapsed from the moment a threat is identified to the moment it is neutralized or mitigated.
- **ARO** - Annual Rate of Occurance refers to the estimated frequency at which a specific type of security incident or threat is likely to occur within a given year.
- **OCSP** - The Online Certificate Status Protocol is an Internet protocol used to check the revocation status of digital certificates in real-time. It is an alternative to Certificate Revocation Lists.
- **PEM** - Privacy-Enhanced Mail, is an Internet standard for secure electronic mail communication. Initially developed to provide security and privacy in email, PEM has evolved to encompass broader cybersecurity applications.
- **SAN** - Storage Area Network refers to a network of storage devices that can be accessed by multiple servers or computers, providing a shared pool of storage space.
- **CRL** - Certificate Revocation List is a digital list maintained by a Certificate Authority (CA) that contains the serial numbers of digital certificates that have been revoked before their scheduled expiration date. Once retrieved, a system will hold a copy of the list in its cache. This cached copy can be used for validation. 
- **OCSP** - An Online Certificate Status Protocol is a cryptographic protocol used to verify the revocation status of digital certificates in real-time. It is an alternative to Certificate Revocation Lists (CRLs) and plays a crucial role in maintaining the security of online communications.
- **MSSP** - Managed Security Service Provider is a type of IT service provider that specializes in offering outsourced monitoring and management of security devices and systems to organizations. Their primary focus is on ensuring the security and integrity of clients’ networks, systems, and data.
- **AUP** - An Acceptable Use Policy is a document that outlines the rules and guidelines for employees or users to follow when using an organization’s computer resources, including networks, systems, and devices.
- **MSA** - Master Service Agreement is a contractual agreement between a service provider (e.g., Managed Security Service Provider, MSSP) and a client (e.g., organization, business) that outlines responsibilities and expectations regarding cyber security.
- **EOL** - End-of-Life refers to software, hardware, or operating systems that are no longer supported by their manufacturers.
- **Detection** - the process of identifying concealed threats, vulnerabilities, or malicious activities within a network, system, or endpoint.
- **CYOD** - Choose your own device is a mobile device management (MDM) strategy that allows employees to select from a pre-approved list of devices provided by the organization for business use.
- **Fuzzing** - Also known as fuzz testing, is an automated software testing method used to identify flaws and vulnerabilities in software applications. It involves feeding a wide range of invalid and unexpected data, known as “fuzz.”
- **SLA** - Service Level Agreement for cybersecurity defines the expected level of service, performance, and response times for cybersecurity-related activities, such as incident response, threat detection, and vulnerability management.
- **UTM** - Unified Threat Management is a comprehensive cybersecurity approach that consolidates multiple security functions into a single system or appliance.
- **NGFW** - Next-Generation Firewall is a network security device that evolves beyond traditional firewalls by incorporating advanced features and capabilities.
- **SCAP** - Security Content Automation Protocol is a set of open standards and specifications that enables the automation of various cybersecurity functions, including vulnerability management, measurement, and policy compliance evaluation.
- **CVE** - Common vulnerabilities and exposures is a standardized system used to identify and track publicly known cybersecurity vulnerabilities and exposures in software, hardware, and digital systems. It provides a unique identifier for each vulnerability, enabling effective communication and coordination among organizations and individuals.
- **WPA3** - Wi-Fi Protected Access 3 is a wireless network security protocol developed by the Wi-Fi Alliance, designed to provide enhanced security and privacy for personal and open Wi-Fi networks.
- **8021.x** - A network authentication standard developed by the Institute of Electrical and Electronics Engineers (IEEE) to ensure secure access to wired and wireless networks. It provides a framework for port-based network access control (PNAC), enabling organizations to authenticate and authorize devices before granting them access to the network.
- **Typosquatting** -  A type of cybercrime that involves registering domain names with deliberately misspelled names of well-known websites, with the intention of luring unsuspecting visitors to alternative websites, typically for malicious purposes.
- **WAF** - A Web Application Firewall is a cybersecurity solution designed to protect web applications from a wide range of cyber threats and attacks. It operates at the application layer.
- **DRP** - A Disaster Recovery Plan (DRP) in the context of cybersecurity refers to a documented process or set of procedures to recover and protect a business IT infrastructure in the event of a disaster
- **Bastion Host** - a specialized computer designed and configured to withstand cyberattacks. Its primary purpose is to prevent malicious traffic from entering a network by processing and filtering all incoming traffic.
- **Digital Forensics** - A branch of forensic science that deals with the acquisition, preservation, analysis, and reporting of digital evidence in a manner that is scientifically reliable, accurate, and repeatable.
